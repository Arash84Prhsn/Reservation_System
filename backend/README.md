# Reservation system backend and logic
This file is meant to document the APIs and the functionality of the backend for the reservation
system website.

## Directories
1. `db`: Stores files concerned with the overall structure of the database.
2. `models`: Stores files concerned with the ORM system and functions for ease of database manipluation.
3. `routes`: Stores files regarding the routes of the website, This is where the main API is.
4. `services`: Contains python scripts that contain functions meant to implement algorithms and to
be used in the `routes` scripts.
5. `utils`: Contains utility scripts meant to help all around.

## API refrence
Below here you will find API refrences for each of the scripts in the routes folder. This is meant 
for the frontend developers so they know exactly what data structures to send and recieve(and how
the data looks in general).
